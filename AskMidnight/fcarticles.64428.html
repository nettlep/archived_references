Windows NT Or 9x
Question submitted by Anonymous (10 July 2000)
 
  What do you recommend Windows coders use, NT or 9x?  
 

 
  Now that Windows2000 is out (with full DirectX support) I highly recommend dumping NT and 9x and moving straight for 2000. Do not pass Go, do not collect $200.

You might have problems getting proper drivers (I still can't get drivers for my digital camera) and some of the NT4 drivers don't work as well as you would hope, but you can't match the stability of 2000 itself unless you go with Linux (and it stings just a little to say that, being a Linux fanatic myself. :)

   Re: Issues that games have with Win2K
The only real issue I see it nVidia's driver hanging the whole machine while using DX (one more reason to hate DX, as OGL works 100% fine :)
Games are slower than 98 because 2k is more restrictive (strictly coupled with stability), has a different CPU-allocation scheduler and does allow some 9x-tricks to happen at high fees. Other cheats are not allowed, and hence the inability to run the code.

It's mainly a problem of rewriting chunks of (hevily-optimized, non-100%-API-compliant) code, and making it run faster.
Plus, the drivers are still under developement, which sure doesn't help :) But, this will change (I hope ;)

   Win2K issue
There is another issue I know of. Windows 2000 sets registers to different values at application's entry point. While this doesn't cause any problems with properly written applications, many PE compressors rely on these values and fail to run. There is even a commercial cryptor which doesn't run on Win2K.
